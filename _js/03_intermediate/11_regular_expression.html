regular Expression : 정규표현식, 줄여서 정규식 이라고 칭함, regExp라는 단어로 많이 표현됨

  • 특정 데이터의 탐색, 확인, 존재여부 등을 쉽게 수행하기 위해 만든 규칙
  • 정규식이 작동되는 전용 메서드에 규칙적인 부호를 사용
  • 일반적인 웹 분야에서는 사용률이 낮지만 데이터 분석과 관련된 분야는 마스터를 해야 하는 기술
  • 알아두면 복잡한 연산을 쉽게 할 수 있어서 시간날 때 학습하길 권장함
  • 프로그래밍 언어별로 조금씩 차이가 있지만 거의 비슷하므로 한번 학습해 놓으면 유용하게 사용할 수 있음
  • Modifier : 제한자 : i/ g /m

JavaScript Regular Expressions

Do a multiline search for "is" at the beginning of each line in a string:

Expression

  1. strData.match(/찾는문자/modifiers) : 찾는 문자와 일치하는 값을 탐색 후 객체리턴 배열과 모습이 같음
  2. strData.match(/[찾는문자]/modifiers) : []안에 찾고자 하는 알파벳 혹은 숫자를 넣을 수 있다.=> 알파벳이나 단일 수로 탐색
  3. [0-9] : 0부터 9까지 해당하는 숫자: -(하이픈)은 ?부터 ?까지 범위를 지정할 때 사용하는 기호 => 즉[0-9]는 모든 숫자를 의미한 => 문자도 사용 가능
  4. [a-zA-Z0-9] : 영문자와 숫자만 가능
  5. (str1 | str2) : str1이나 str2에 정확하게 부합하는 데이터 탐색

MetaCharacter

  • 브라켓을 사용하는 표현식을 단순하게 특정 문자로 대체한 개념
  • \d:숫자만, \w: 문자만 etc.
  • 자주사용되는 메타캐릭터들은 기억해 두면 좋다 => 매우 유용하게 사용됨

Quantifier : 수량결정자, 수량제한자, 수치표현자

  1. n+ : n을 포함하고 n뒤로 최소1개 이상의 문자
  2. n*: n을 포함하고 n뒤로 0개 이상의 문자
  3. n? : n을 포함하고 n뒤로 1개의 문자
  4. \d{3} : 3자리 digit character (숫자모양을 한 문자)
  5. \d{3,4} : 3 or 4자리 digit character (숫자모양을 한 문자)
  6. \d{3,} : 3자리이상 digit character (숫자모양을 한 문자)
  7. ^word : word로 시작하는 단어 => []안에서 사용하면 not의 의미가 됨
  8. word$ : wordf로 끝나는 단어
  9. word(?=str) : word 뒤에 1개 이상 str이 오는 word를 리턴
  10. word(?! str) : word 뒤에 1개 이상 str이 오지 않는 word를 리턴

regExp Object Methods

  1. pattern.test('strValue') : 해당 패턴이 맞는지 여부 : 리턴은 boolean
  2. pattern.exec('strValue') : 해당 패턴을 값에 대입하여 실행 : 리턴은 객체(match메서드와 같음)=> 객체의 프로퍼티에 접근하면 각 결과값을 알 수 있음
